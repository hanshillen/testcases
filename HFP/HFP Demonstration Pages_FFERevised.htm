JAWS Test Page: Reading Form Controls

NOTE: 12/22/2010 ESW - I called FreedomScientific about the differences between the Read All and Read line by line commands. FS agreed with my findings based on if data was present within textboxes, textareas, etc. As per FS, form elements, with data not present, are read for example "Name 1:" with the Read All command. The form element type is relayed with the read line by line feature because forms mode is on when in this mode, regardless if data is present in the form element or not.

NOTE: When navigating the page with JAWS either using Tab or pressing the "F" key, the user is navigated to the form element on the page. JAWS reads all legend, label, etc. correctly also conveying the type of form element selected (i.e. Name 1: Edit). A noteworthy difference is a textbox, navigating by pressing "F", is read as "Name 1: Edit" yet a textarea with data within is read as "Enter your address 1: contains text" not reading the data within the text area. All other form elements are read the same across all versions of JAWS that we use.
